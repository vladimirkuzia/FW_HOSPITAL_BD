## Общая схема БД:
![Alt-ERD for database](images/structbd.png)

Использование интуитивно понятное, БД больница, может применятся:

**Администраторами клиники:**

*Управление информацией о пациентах, врачах и медицинских записях.

*Генерация отчетов о работе клиники.

**Врачами:**

1.Просмотр и обновление медицинских записей пациентов.

2.Выписка рецептов.

3.Просмотр записей.

**Пациентами:**

1.Просмотр информации о своих записях к врачам и рецептах.

2.Запись на прием к врачу.

P.S.*На многих атрибутах есть ограничения NOT NULL, каждая из таблиц содержит PRIMARY KEY, в БД используются внешние ключи для организации отношений между таблицами.*

## Для взаимодействия с данной БД используйте скрит hospital_programm.py:
````
python3 hospital_programm.py
````
Данная программа представляет собой личный кабинет для врача, логин и пароль генерируются автоматически в самой БД. Для проверки, необходимо взять их из таблицы doctorautor. В перечне возможных действий : Просмотреть записи на сегодняшний день, Добавить медицинскую запись, Добавить рецепт, Добавить запись пациента, Выйти.

*Некоторые действия могут вызвать ошибку, такие как: неверный логин/пароль, запись мужчины к врачу-гинекологу, запись пациента на время, где уже стоит другая запись.*

**Список созданных процедур, функций, представлений, триггеров с их описанием, параметрами работы**

**Процедуры:**

*add_appointment* – добавление в таблицу appointment записи пациента к врачу

*add_doctor* - добавление в таблицу doctors доктора

*add_patient* -  добавление в таблицу patients пациента

*update_doctor_departments* –  добавляет в таблицу doctors department_id, сверяя поля специализации врача и названия отдела из таблицы departments

*add_medical_records* - Запись в таблицу добавляется для того врача и пациента, у которых в таблице appointments есть встреча со статусом "Завершено", поэтому на поле record_date ограничение, дата(год и число) должна соответствовать той, что в таблице appointments

*add_prescription* - Добавляет рецепт, если медицинская запись существует и статус обследования "Завершено"

**Триггеры:**

*check_appointment_gender* – проверка на то, что к Урологу могут записаться только мужчины, а к Гинекологу – только женщины

*check_appointment_time* – проверка нет ли уже других записей на это время и не проходит ли в это время другая запись, запланированные записи в 1 день у 1 врача должны быть с промежутком в 30 минут

*update_patient_age* – по полю даты рождения запишет cколько лет пациенту

**Функция:**

*generate_random_password* – генерирует рандомный пароль для докторов из 20 символов
